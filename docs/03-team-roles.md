## 1. 팀 구성

> 실제 팀원 이름·이메일은 노션 페이지·내부 채팅에 별도 관리. 본 문서는 **역할 기준 R&R**만 다룬다.

| 역할 | 인원 | 주요 책임 |
|---|---|---|
| **백엔드 리드** | 1명 | FastAPI 서버·DB·인증·게이미피케이션·테스트 |
| **프론트 리드** | 1명 | React/Vite UI·디자인 시스템·시연 흐름 |
| **AI/ML 리드** | 1명 | Model 1·2 학습·평가·OCR·LLM 인터페이스 설계 |
| **기획·발표** | 1명 | 요구사항 정의서·발표 자료·시연 시나리오 |
| **(팀 리더·통합)** | 1명 | 일정·미팅 진행·외부 조율·문서 통합 |

## 3. 협업 도구·체계

| 도구 | 용도 |
|---|---|
| **GitHub** | 코드 저장소·PR 리뷰·이슈 추적 |
| **GitHub Actions (CI)** | 자동 lint·test |
| **Notion** | 회의록·기획·DB 정의·R&R 상세 |
| **Slack/카카오톡** | 일상 커뮤니케이션 |
| **Jira/Linear** (선택) | 스프린트·티켓 |

### 코드 리뷰 정책

- PR 머지 전 **최소 1인 리뷰**
- 의료 콘텐츠 변경 시 `medical-review` 라벨 필수
- 개인정보 흐름 변경 시 `privacy-review` 라벨 필수
- Squash merge (히스토리 단순화)

### 미팅 주기

- **일일 스탠드업**: 평일 오전 (15분)
- **주간 회고**: 매주 금 (1시간) — 진행 점검·우선순위 조정
- **마일스톤 회의**: 6/12 배포 / 6/19 발표 전 1회씩

## 5. 책임 매트릭스 (RACI)

| 작업 영역 | 백엔드 | 프론트 | AI | 기획 | 통합 |
|---|---|---|---|---|---|
| 백엔드 API 구현 | **R** | C | C | I | A |
| 프론트 UI 구현 | C | **R** | I | C | A |
| ML 모델 학습 | C | I | **R** | I | A |
| 요구사항 정의 | C | C | C | **R** | A |
| 발표 자료 | I | C | C | **R** | A |
| 시연 시나리오 | C | C | C | **R** | A |
| 의료법 준수 | **R** | **R** | C | C | A |
| 일정 관리 | I | I | I | C | **R** |
| 코드 리뷰 | **R** | **R** | **R** | I | A |
| 배포 | **R** | C | C | I | A |

> R = Responsible (실행), A = Accountable (최종 책임), C = Consulted (자문), I = Informed (보고)
