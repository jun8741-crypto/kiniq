## 1. 왜 게이미피케이션을 추가했나

### 우리 서비스의 문제

CKD(만성 신장질환) 관리는 **수개월~수년 단위 장기전**입니다. 사용자가 매일 챌린지(물 마시기, 저염 식단 등)를 체크인해야 하는데, 의학적 효과는 천천히 나타나기 때문에 **단기 동기 부여가 약합니다.**

→ 사용자가 며칠 만에 이탈할 위험이 큼.

### 게이미피케이션의 역할

심리학·행동경제학 기반 메커니즘으로 **"오늘도 들어올 이유"**를 만든다:

| 메커니즘 | 우리 구현 | 효과 |
|---|---|---|
| **변동 보상 (Variable Reward)** | 럭키 체크인 10% 확률로 포인트 ×2 | 매번 체크인이 "혹시?" 기대감 |
| **목표 근접 효과 (Goal Gradient)** | 알 부화 진행률 70%·90% 도달 시 알림 | "조금만 더!" 마무리 욕구 자극 |
| **수집욕 (Collection)** | 부화 후 3종 캐릭터 컬렉션 | "다음엔 다른 종 나왔으면" |
| **손실 회피 (Loss Aversion)** | 스트릭(연속 출석) 일수 표시 | "깨고 싶지 않다" 심리 |
| **부드러운 회복 (Soft Failure)** | 쉬어가기 모드 (강등·실패 단어 금지) | "낙오자 만들지 않기" |

### 의학적 안전성 보장

- **벌점·강등 단어 금지** — 고령 사용자 비율 고려, 부정적 자극 회피
- **포인트는 게임 내 화폐만** — 현실 화폐·실제 상품 교환 없음 (의료 광고법 회피)
- **챌린지 자체는 의학 기반** — 게이미피케이션은 "동기 부여 레이어", 실제 행동은 의학적으로 안전한 것만

## 2. 핵심 개념 정리

### 한 줄 정리

> 사용자가 챌린지를 체크인할 때마다 **포인트**가 쌓이고, **알**이 자라며, **부화하면 캐릭터**가 나오고, 사용자는 포인트로 **아이템**을 사거나 캐릭터에게 **스킨**을 입힐 수 있다.

### 6가지 핵심 요소

| 요소 | 무엇 |
|---|---|
| 🪙 **포인트** | 행동 보상으로 적립되는 게임 내 화폐. 정수. 만료 없음 (v1.0). |
| 🥚 **알** | 사용자가 키우는 가상 대상. 체크인 100회 누적 시 부화. |
| 🐢 **캐릭터** | 부화 시 나오는 동물. 거북이·펭귄·다람쥐 3종. |
| 🛡️ **보호권** | 깜빡한 날 자동 사용되어 스트릭 보존. |
| 🎨 **스킨** | 캐릭터 외형 변경. 5종. 사용자가 선택 장착. |
| 😴 **쉬어가기 모드** | 7일 미체크인 시 자동 진입하는 부드러운 휴식 상태. |

## 4. 사용자 시나리오 — 5가지 흐름

### 시나리오 ①: 첫 부화 (해피 패스)

```
1. 새 사용자 회원가입 → 로그인 (+10pt, 일일 로그인 보너스)
2. 건강검진 입력 → CKD 단계 G2 자동 판정 → Track A 챌린지 추천
3. 챌린지 3개 참여 (수분/식단/운동)
4. 매일 체크인:
   - 매 체크인 +20pt (10% 확률로 럭키 ×2 = +40pt)
   - 3개 모두 체크인 시 풀 참여 +40pt 보너스
   - 연속 3일 도달 → 스트릭 보너스 +30pt
5. 25일째 (누적 75회 체크인): 알 75% 도달 → 스테이지 보너스 +350pt
6. 32일째 (누적 95회): 90% 알림 — "🔥 부화 임박!"
7. 35일째 (누적 100회): 부화! 🎉
   - 거북이/펭귄/다람쥐 중 랜덤 추첨
   - "고결한 파랑" 같은 이름 자동 생성
   - 부화 보너스 +600pt
8. 컬렉션 페이지에서 새 캐릭터 확인 → 이름 수정 가능
9. 새 알이 자동으로 시작 (egg_no=2, 0%부터)
```

### 시나리오 ②: 럭키 체크인 발동

```
1. 사용자가 체크인 버튼 클릭
2. 백엔드가 random.random() 시행 → 0.05 (10% 미만)
3. 럭키 발동! 기본 +20 → +40으로 적립
4. 프론트에 결과 모달:
   ┌────────────────────────┐
   │      ✨                │
   │  럭키 체크인!          │
   │  포인트가 두 배로!     │
   │                        │
   │  체크인     +20pt      │
   │  럭키 ×2    +20pt      │
   │  합계       +40pt      │
   └────────────────────────┘
5. 컨페티 60개가 화면에 떨어짐 (2.5초 애니메이션)
```

### 시나리오 ③: 깜빡한 날 + 보호권 보유

```
[월~목] 사용자 체크인 4일 연속 (streak=4)
[금]    까먹음! 체크인 0개
[토]    앱 열고 로그인
  → 서버: "어제 체크인 0개네? 보호권 있어?" → 있음
  → 보호권 1개 차감, 가상 체크인으로 last_checkin_date=금요일 설정
  → 알림: "스트릭 보호권이 사용됐어요"
[토] 사용자가 체크인 → streak=5 유지! 끊김 없음
```

### 시나리오 ④: 며칠 안 들어옴 → 쉬어가기 모드

```
[5/20] 마지막 체크인
[5/24] 4일째 — 알림: "4일째 쉬는 중이에요. 지금 1번만 체크인하면 정상으로 돌아갑니다."
[5/25] 5일째 — 동일 알림
[5/26] 6일째 — 동일 알림 (마지막 경고)
[5/27] 7일째 — 자동으로 쉬어가기 모드 진입
  → 알림: "쉬어가기 모드 시작. 잠시 쉬는 동안 스트릭은 보존됩니다."
  → 부스터 보유 시: 9일째로 진입 늦춰짐 + 부스터 1개 자동 소모
[5/30] 사용자 돌아옴 → 체크인 1회 → 즉시 정상 모드 복귀
```

### 시나리오 ⑤: 스킨 구매 + 장착

```
1. 사용자 10,000pt 보유
2. 상점 → 골드 스킨 (대) 1,200pt 구매
3. 잔액 8,800pt + 인벤토리에 SKIN_L_GOLD 1개 추가
4. 컬렉션 페이지로 이동
5. 보유 스킨 목록에서 "골드 스킨 (대)" 카드의 [장착] 클릭
6. User.active_skin_code = "SKIN_L_GOLD" 저장
7. 카드에 "장착 중" 배지 표시
8. 대시보드 EggWidget이 갱신되며 스킨 효과 반영 (시즌 한정 표시 등)
```

| 풀 참여 보너스 | +40 | 그 날 active 챌린지 전부 체크인 |
| 스트릭 3일 | +30 | 마일스톤 도달 시 1회 |
| 스트릭 7일 | +70 | 마일스톤 도달 시 1회 |
| 스트릭 14일 | +150 | 마일스톤 도달 시 1회 |
| 스트릭 30일 | +300 | 마일스톤 도달 시 1회 |
| 알 25% 도달 | +100 | 1회 |
| 알 50% 도달 | +200 | 1회 |
| 알 75% 도달 | +350 | 1회 |
| 알 100% 부화 | +600 | 1회 |

|---|---|

## 6. 알 부화 시스템 상세

### 진행률 계산

**누적 체크인 = 진행률**. 체크인 100회로 부화. 매 체크인마다 +1.

### 5단계 시각화

| 단계 | 진행률 | 이모지 | 의미 |
|---|---|---|---|
| 1 | 0~24% | 🥚 | 갓 시작 |
| 2 | 25~49% | 🥚 | 자라는 알 (배경 색상 변화) |
| 3 | 50~74% | 🐣 | 큰 알 |
| 4 | 75~99% | 🐥 | 부화 직전 |
| 5 | 100% | 🐤→🎉 | 부화 완료! |

### Goal Gradient 알림 (목표 근접 효과)

심리학적으로 **목표에 가까워질수록 동기가 강해진다**는 효과를 이용:

- **70% 도달 시**: 알림 1회 — "알이 거의 자랐어요. 부화까지 30번만 남았어요."
- **90% 도달 시**: 알림 1회 — "부화 임박! 10번만 더!"

→ 결정적 순간에 "조금만 더!" 자극으로 매일 들어오게 유도

### 부화 시 처리

```
1. 100% 도달 감지
2. 종 추첨: random.random() → 0~1 사이 균등 분포
   - 0.0~0.333 → TURTLE 🐢
   - 0.333~0.666 → PENGUIN 🐧
   - 0.666~1.0 → SQUIRREL 🐿️
3. 자동 이름 생성: 형용사 12개 × 종별 별칭 5개 = 60가지 조합
   예: "고결한 파랑", "호기심 많은 차마", "용맹한 찌이"
4. UserEgg.hatched_at = 현재 시각 기록
5. 새 UserEgg 생성 (egg_no+1, progress=0)
6. 부화 알림 발송: "🐢 거북이 부화! '고결한 파랑' 가 태어났어요."
7. CheckinResultModal에 큰 캐릭터 이모지 + 이름 표시
```

## 7. 3종 캐릭터 시스템

### 캐릭터 종

| 종 | 이모지 | 라벨 | 모티브 |
|---|---|---|---|
| TURTLE | 🐢 | 거북이 | "고결한 파랑" (네이버 옐로우마을 캐릭터 오마주) |
| PENGUIN | 🐧 | 펭귄 | "호기심 많은 차마" |
| SQUIRREL | 🐿️ | 다람쥐 | "용맹한 찌이" |

### 자동 이름 생성

**형용사 12개:**
고결한, 호기심 많은, 용맹한, 재빠른, 온화한, 장난스러운, 당당한, 총명한, 포근한, 씩씩한, 느긋한, 꾸준한

**종별 별칭 5개씩:**
- 거북이: 파랑, 또롱, 거북이, 퍼북, 구이
- 펭귄: 차마, 포포, 펭귄, 쮸쮸, 콩이
- 다람쥐: 찌이, 쪼롱, 다람, 도토리, 꼬리

→ **총 12 × 5 = 60가지 이름 조합** 가능

### 사용자가 이름 수정 가능

부화 후 컬렉션 페이지에서 ✏️ 아이콘 클릭 → 인라인 입력 → ✓ 저장.

API: `PATCH /api/v1/gamification/eggs/{id}/name`

### 캐릭터별 차이

**외형 + 이름만 다름.** 능력치·보너스 동일.

이유: 게임 밸런스 단순화. 차이를 두면 "어느 종이 더 강한가?" 메타가 생겨서 사용자가 자기 종을 불만스러워할 수 있음. 외형 컬렉션 욕구만 자극.

## 8. 스트릭과 보호권

### 스트릭(연속 출석) 계산 규칙

```python
오늘 체크인할 때:
  if 어제 last_checkin_date == 어제:
      streak += 1  # 연속!
  else:
      streak = 1   # 끊김, 1부터 다시
```

### 보호권 자동 소모 (구현됨)

**조건 4가지 모두 충족 시 자동 차감:**

1. 어제 active 챌린지에 체크인 0회
2. 어제가 챌린지 시작일보다 이후 (= 어제 보호 대상 챌린지가 있었음)
3. 보호권 1개 이상 보유
4. 같은 날짜에 대한 PROTECT_CONSUME 트랜잭션이 아직 없음 (멱등)

**동작:**
- 보호권 1개 차감
- 어제 날짜로 **가상 체크인 기록** (각 active 챌린지의 last_checkin_date = yesterday)
- 오늘 실제 체크인 시 자연스럽게 streak +1
- 알림 발송: "스트릭 보호권이 사용됐어요"

**평가 시점**: 사용자가 로그인 또는 체크인할 때 lazy 평가 (cron 없음).

### 보호권 보유 한도

**최대 2개**. 이유:
- 무한 보유 허용 시 → 사용자가 포인트 쌓아서 보호권 도배 → 실제 행동 없이 알만 키우는 패턴 발생
- 2개 캡이 "비워서 채우는" 소비 루프 형성

## 9. 쉬어가기 모드 (충전 모드)

### 진입 조건

**기본 7일** 동안 체크인 0회 시 자동 진입. **부스터 보유 시 9일**로 연장 + 부스터 1개 자동 소모.

### 경고 알림 (4·5·6일째)

```
4일째: "4일째 쉬는 중이에요. 지금 1번만 체크인하면 정상으로 돌아갑니다."
5일째: 동일
6일째: 동일 (마지막)
7일째: 자동 진입 (또는 9일째 if 부스터 있음)
```

### 진입 효과

- `UserChargeMode.is_active = True`, `entered_at` 기록
- **스트릭 동결** — 진입 동안 0으로 리셋되지 않음
- 알 진행률은 그대로 유지
- 대시보드 EggWidget에 회색 "쉬어가기 모드" 배너 표시 → `/rest-mode` 페이지 링크

### 탈출

**체크인 1회로 즉시 정상화**. 또는 `/rest-mode` 페이지의 "체크인 없이 즉시 탈출" 버튼.

### 언어 대체표 (절대 준수)

| ❌ 금지 | ✅ 사용 |
|---|---|
| 강등 | 쉬어가기 모드 |
| 레벨 다운 | 충전 모드 |
| 실패 | 잠시 멈춤 |
| 벌점 | 단어 자체 금지 |

이유: **고령 사용자 비율**과 **만성질환 관리 부담 사용자** 고려. 부정적 자극이 이탈 유발.

## 12. 팀 합의가 필요한 결정 사항

> ⚠️ **풀스택이 단독으로 구현했으나 팀 합의는 아직 없음.** 합의 결과에 따라 일부 코드 재작업 가능성.

### 결정 #5 — 알 부화 트리거 기준

**현재 구현 (잠금판):**
- 부화 목표: 누적 체크인 **100회**
- 5단계 임계: 25 / 50 / 75 / 100%
- Goal Gradient: 70% / 90% 알림

**합의 필요:**
- 100회가 적절한가? (너무 길면 사용자 지침, 너무 짧으면 보상 가치 하락)
- 70%/90% 외에 다른 시점도? (예: 95%·99%)

### 결정 #7 — 충전 모드 + 미니알 회복

**현재 구현 (잠금판):**
- 진입 임계: 7일 (부스터 9일)
- 경고: 4·5·6일째
- 탈출: 체크인 1회

**합의 필요:**
- 7일이 너무 관대/엄격한가?
- 부스터 효과를 "진입 지연"이 아니라 "탈출 가속"으로 바꿔야 하나?

### 캐릭터 시스템 — 전설 5% 제거

**현재 구현 (잠금판):**
- 부화 시 3종 균등 추첨 (33.33%)
- **전설 5% 추첨 제거** (일러스트 부담)

**합의 필요:**
- 전설을 v1.0에 다시 넣을지? 일러스트 누가 그릴지?
- 종 추첨 비율을 균등이 아니라 가중치(예: 거북이 50%)로?
