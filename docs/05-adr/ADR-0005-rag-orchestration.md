# ADR-0005: RAG 오케스트레이션 — LangGraph (Self-RAG)

**Date**: 2026-06-18
**Status**: ✅ Accepted

## Context

신장 케어 챗봇과 리포트 AI 가이드는 **검색 증강 생성(RAG)**으로 KDIGO·대한신장학회 자료에 근거한 답변을 만든다. 단순 "검색 → 생성" 1패스로는 다음을 보장할 수 없다:

- 검색된 청크가 질문과 **무관할 때 재검색/재작성**(Self-RAG)
- 의료 도메인 **안전 가드**(약물 용량·진단 단정 차단)와 답변 재생성
- 조건부 분기(검색 품질 평가 → 통과/재시도)와 상태 누적이 있는 **제어 흐름**

즉 선형 체인이 아니라 **조건부 그래프**가 필요하다.

## Decision

**LangGraph `StateGraph` 기반 Self-RAG 2단계** 파이프라인 채택.

- 노드: `retrieve → grade(검색 적합성 평가) → generate → (안전/근거 미흡 시) rewrite·재검색`
- 생성·평가 LLM은 **temperature=0**으로 고정(결과 편차 최소화, 평가 3-3)
- 토큰 스트리밍(SSE)과 결합해 최종 답변만 사용자에게 노출

## Alternatives Considered

| 후보 | 장점 | 단점 | 기각 사유 |
|---|---|---|---|
| LangChain 단순 체인(LCEL) | 간단, 진입 쉬움 | 조건부 분기·재시도·상태 누적 표현 약함 | Self-RAG 제어 흐름 부적합 |
| LlamaIndex | 인덱싱·쿼리 엔진 강점 | 멀티스텝 워크플로우 제어는 보조적 | 그래프형 제어가 1급이 아님 |
| 직접 구현(상태머신) | 의존성 0, 완전 제어 | 분기·재시도·체크포인트를 전부 자작 | 유지보수 비용·재현성 보장 어려움 |
| **LangGraph** ⭐ | 그래프 제어 1급, 조건부 엣지·상태 채널, LangChain 생태계 재사용 | 러닝커브, 디버깅 시 그래프 추적 필요 | — (선택) |

## Consequences

### 좋은 점
- `grade` 노드가 검색 품질을 평가해 **근거 없는 답변(hallucination)을 재검색/재작성**으로 차단
- 상태 채널로 user_context(eGFR·위험군·식이 플래그)를 그래프 전역에 일관 주입
- temperature=0 + 산출물 동결로 **동일 질문 결과 편차 최소화**(평가 3-3 연계)

### 트레이드오프
- 그래프 정의·디버깅 러닝커브 (LCEL 대비 코드량 ↑)
- 멀티 노드라 단일 추론 지연이 늘 수 있음 → 비동기 워커 위임(ADR-0007)·가이드 사전 생성 캐시로 상쇄

### 운영 영향
- RAG 추론은 `ai_worker/rag`에서 실행, 백엔드와 Redis Streams로 디커플(ADR-0007)
- 벡터 검색은 Qdrant(ADR-0006)
