## 0. 용어 정리

- **포인트(point)**: 행동 보상으로 적립되는 게임 내 화폐. 정수.
- **알(egg)**: 사용자가 키우는 가상 대상. 100% 도달 시 부화하고 새 알이 시작됨.
- **스테이지(stage)**: 알의 성장 단계. 1(0~25%) → 2(25~50%) → 3(50~75%) → 4(75~100%) → 5(부화).
- **스트릭(streak)**: 연속 체크인 일수.
- **럭키 체크인**: 10% 확률로 그 체크인의 기본 보상이 ×2 되는 가변 보상.
- **충전 모드(쉬어가기 모드)**: 일정 기간 미체크인 시 진입하는 부드러운 회복 상태. **"강등" 단어 금지.**
- **레벨 다운**: 챌린지 트랙 강등(A → B). 본 문서 범위 밖, REQ-CHAL-005 별도 추후.

## 1. 잠금 파라미터

### 1-1. 포인트 적립 (PointService)

| 행동 | 포인트 | 트리거 |
|---|---|---|
| 일일 로그인 (당일 첫 인증 요청) | +10 | `/auth/login` 또는 첫 인증된 요청 |
| 챌린지 체크인 1개 | +20 (럭키 시 +40) | `POST /user-challenges/{id}/checkin` 성공 시 |
| 풀 참여 보너스 | +40 | 그 날 active 챌린지 전부 체크인 완료 시점에 1회 |
| 스트릭 보너스 3/7/14/30일 | +30 / +70 / +150 / +300 | 스트릭 카운트가 해당 값에 **처음 도달한 그 날** |
| 부화 (1단계 캐릭터 등장) | +100 | 누적 10회 체크인 도달 시 1회 |
| 2단계 진화 | +400 | 누적 40회 체크인 도달 시 1회 |
| 3단계 최종 진화 (완전체) | +750 | 누적 100회 체크인 도달 시 1회 (완료, 추가 알 없음) |

### 1-2. 럭키 체크인

- **적용 단위**: 매 체크인 1회마다 독립 시행
- **확률**: 10% (`random.random() < 0.10`)
- **적용 범위**: 기본 체크인 보상 +20 → +40 만. 보너스(풀 참여·스트릭·스테이지)는 영향 없음
- **알림**: 럭키 발동 시 응답에 `lucky: true` 플래그 포함, 프론트가 컨페티 등 시각 효과 노출

### 1-3. 알 → 캐릭터 3단계 진화 (EggService, v3 갱신)

> v2(4단계, 2026-05-27) → **v3 단순화 3단계** (2026-06-01 사용자 요청). 캐릭터 형태는 부화 1번 + 진화 2번 = 1·2·3 세 가지.

- **단계 전환 임계 (누적 체크인)**:
  - 0~9회: 알 🥚 (stage=0)
  - **10회 도달 → 부화 (1단계 캐릭터 등장)**, 종 추첨 + 자동 이름 생성
  - **40회 도달 → 2단계 진화**
  - **100회 도달 → 3단계 최종 진화 (완전체, 완료)**
- **간격 디자인**: 10 → 30 → 60 (초반 빠른 부화, 중후반 점진적 장기 목표)
- **보너스 분배 (합 1,250pt)**: +100 / +400 / +750 — Goal Gradient 효과 유지하면서 후반 강화
- **완료 후 처리**: 3단계 도달 시 **새 알 자동 시작 X** (단일 캐릭터로 완료). 추가 체크인 시 진행률 freeze, EggService는 효과 없음 반환.
- **전설 추첨**: v1.0 비활성. `is_legendary` 컬럼은 유지(항상 False), 향후 부활 여지.
- **Goal Gradient 알림**: 90회 (3단계 임박 90%) **첫 도달** 시 알림 1회.
- **이미지 자산**: `frontend/.../public/characters/` — SVG 10장(egg + 종3 × stage3) 동봉. 사용자 직접 PNG 교체 시 동일 파일명. fallback: 종 이모지 (🐢🐧🐿️) + 단계별 fallback (🥚 → 🐣 → 🐥 → 🌟).

### 1-4. 인벤토리 / 소비 (InventoryService)

| 아이템 | 비용 | 최대 보유 | 효과 |
|---|---|---|---|
| 스트릭 보호권 | 500pt | **2개** | 체크인 0개인 날 1회 자동 소모해 스트릭 유지 |
| 회복 미니알 부스터 | 200pt | 무제한 | 충전 모드 진입 임계를 7일 → 9일로 +2일 연장 |
| 캐릭터 스킨 (소) | 300pt | 1종당 1개 (중복 구매 차단) | 외형 변경 |
| 캐릭터 스킨 (중) | 700pt | 1종당 1개 | 외형 + 효과 |
| 캐릭터 스킨 (대) | 1,200pt | 1종당 1개 | 외형 + 시즌 한정 표시 (시즌 로직은 v1.1) |

**보호권 자동 소모 우선순위**:
1. 사용자가 그날 active 챌린지 1개 이상에 체크인 0회
2. 그 날 자정 정산 시점 잡(또는 다음날 첫 요청 시 lazy 평가)
3. 보호권 1개 차감, `point_transactions`에 소비 기록, 스트릭 유지

### 1-5. 충전 모드 (쉬어가기 모드)

- **진입 조건**: 7일 연속 체크인 0회 (`last_checkin_at`이 7일 전보다 오래됨)
  - 부스터 보유 시 9일로 연장 (자동 적용)
- **경고 버퍼**: 진입 4·5·6일째 알림 1회씩 — "지금 1번만 체크인하면 정상으로 돌아갑니다"
- **진입 효과**:
  - 스트릭 카운트 임시 보존 (0으로 리셋되지 않음, "잠시 멈춤" 상태)
  - 충전 모드 시작 시각 `entered_at` 저장
  - 알 진행률은 그대로 유지
- **탈출 조건**: 체크인 1회로 즉시 정상화
- **언어 대체표** (UI 카피 절대 준수):
  - "강등" → "쉬어가기 모드"
  - "레벨 다운" → "충전 모드"
  - "실패" → "잠시 멈춤"
  - "벌점" → 단어 자체 금지

### 1-6. 스트릭 정의

- **증가**: 그 날 active 챌린지 중 1개 이상 체크인 완료 시 +1
- **유지**: 보호권 자동 사용 시 유지 (실제 체크인 없어도)
- **리셋**: 0개 체크인 + 보호권도 없는 날 → 다음날 0으로 리셋
- **충전 모드 중**: 동결 (리셋되지 않음, 증가도 안 함)

### 1-7. 포인트 유효기간

- **v1**: 무제한 누적 (만료 없음)
- **v1.1 (발표 후)**: 90일 미사용 시 만료 검토 — 인플레이션 방지

## 2. 데이터 모델 (Tortoise)

### 2-1. `point_transactions`

| 컬럼 | 타입 | 비고 |
|---|---|---|
| id | UUID | PK |
| user_id | FK(users) | |
| amount | INT | + 적립 / − 소비 |
| reason | ENUM | login / checkin / lucky / streak_bonus / stage_bonus / full_participation / purchase / protect_consume / refund |
| metadata | JSONB | 컨텍스트 (challenge_id, stage_no, streak_day, item_code 등) |
| created_at | DATETIME | |

**현재 잔액** = `SUM(amount) GROUP BY user_id`. 트랜잭션 격리 보장을 위해 SELECT FOR UPDATE 사용 또는 user_points 캐시 테이블 검토(v1엔 매번 SUM, 성능 이슈 시 v1.1).

### 2-2. `user_eggs`

| 컬럼 | 타입 | 비고 |
|---|---|---|
| id | UUID | PK |
| user_id | FK(users) | |
| egg_no | INT | 사용자 기준 1, 2, 3 ... (몇 번째 알인지) |
| progress_checkins | INT | 누적 체크인 수 (0~100) |
| current_stage | INT | 1~5 |
| is_legendary | BOOL | 부화 직후 5% 추첨 결과 (현재 알엔 NULL, 부화 후 알엔 true/false) |

| goal_90_alerted | BOOL | 90% 알림 발송 여부 |

**현재 알 조회** = `hatched_at IS NULL`인 row 1개. 부화 시 그 row의 hatched_at 기록 + egg_no+1로 새 row 삽입.
