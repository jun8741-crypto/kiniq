# 콩팥 챌린지 — 기록 기능 기획서

---

## 1. 기능 분류 및 성격 정의

7개 기능은 입력 주기와 데이터 성격에 따라 두 축으로 나뉜다.

**일일 트래킹 (Daily Log)** — 매일 반복 입력, 오늘 하루를 닫는 루틴의 일부
- 수분 섭취 기록
- 체중 기록
- 수면 기록
- 스트레스 감정 기록
- 운동 피로도 기록

**아카이브 (Archive)** — 비정기적 누적, 의료진과 공유하거나 추세를 확인하는 목적
- 병원 진료일 캘린더
- 검사 수치 기록장

이 구분은 UI 배치 결정의 근거가 된다. 일일 트래킹은 오늘 탭의 하단 또는 별도 기록 탭에 위치하고, 아카이브는 별도 탭으로 분리한다.

---

## 2. 기능별 상세 기획

---

### 2-1. 수분 섭취 기록

**목적**
투석·CKD 트랙 사용자는 수분 제한이 처방되어 있다. 실시간 누적량을 확인함으로써 처방 한도를 넘기지 않도록 한다. 집중케어·일상케어·웰니스 트랙 사용자는 반대로 목표 섭취량 달성을 확인하는 데 사용한다.

**입력 방식**
음료 종류별 빠른 추가 버튼(물, 커피, 주스, 기타) + 직접 입력(mL). 한 번에 100mL / 150mL / 200mL / 250mL 단위 버튼으로 빠르게 추가. 수정 및 삭제 가능.

**표시 정보**
- 오늘 누적 섭취량 (mL)
- 목표 대비 진행률 (원형 게이지 또는 바)
- 목표량: 트랙·처방에 따라 사용자가 직접 설정 (투석 트랙은 제한량, 그 외는 권장량)
- 섭취 내역 리스트 (시간 + 종류 + 용량)

**트랙별 목표 기본값**

| 트랙 | 기본값 | 설정 방향 |
|------|--------|-----------|
| 투석 트랙 | 처방에 따라 다름 (예: 500~1,000mL) | 상한 경고 |
| 비투석 CKD | 의료진 지시에 따라 | 상한 경고 |
| 집중케어·일상케어 | 2,000mL | 목표 달성 유도 |
| 웰니스 | 2,000mL | 목표 달성 유도 |

**경고 조건**
투석·CKD 트랙에서 설정 목표의 90% 도달 시 시각적 경고(색상 변환). 100% 초과 시 알림.

**데이터 보존**
날짜별 저장. 최근 30일 내역 조회 가능.

---

### 2-2. 체중 기록

**목적**
수분이 체내에 과잉 축적되는지를 매일 아침 같은 조건(기상 직후, 소변 후, 식전)에서 체중을 측정해 전날 대비 변화를 확인한다. 투석 트랙에서는 투석 간 체중 증가 한도 확인에 직접 사용된다.

**입력 방식**
숫자 입력 (소수점 1자리, kg). 측정 시각 자동 기록 (입력 시점). 메모 필드 선택적 추가.

**표시 정보**
- 오늘 체중
- 어제 대비 증감 (▲/▼ + kg)
- 최근 7일 미니 트렌드 차트 (꺾은선)
- 투석 트랙 전용: 지난 투석 이후 증가량 표시 (사용자가 투석일을 지정하거나 캘린더와 연동)

**경고 조건**
전일 대비 1kg 이상 증가 시 주의 표시. 2kg 이상 시 "의료진에게 확인하세요" 경고 문구 표시.
(투석 트랙 기준. 다른 트랙은 해당 없음 또는 별도 설정)

**데이터 보존**
날짜별 1회 저장 원칙 (수정 가능). 최근 90일 내역 조회.

---

### 2-3. 수면 기록

**목적**
수면 시간과 규칙성이 혈압·혈당·신장 기능에 영향을 준다는 근거에 따라, 취침 시각과 기상 시각을 기록하고 수면 시간을 자동 계산한다.

**입력 방식**
취침 시각 입력 + 기상 시각 입력 (time picker). 전날 밤 취침 → 오늘 아침 기상으로 하나의 수면 세션을 구성한다. 수면 중 깬 횟수 선택 입력 (0~3회 이상).

**표시 정보**
- 오늘 수면 시간 (자동 계산, 시간 단위)
- 취침 시각 / 기상 시각
- 7시간 이상 여부 판정 (달성 / 미달)
- 최근 7일 수면 시간 바 차트
- 평균 취침 시각 / 기상 시각 (7일)

**데이터 보존**
날짜별 저장. 최근 30일 내역.

---

### 2-4. 스트레스 감정 기록 — 감정 쓰레기통

**목적**
스트레스 상황의 감정을 글로 쓰고 '버리는' 행위를 통해 심리적 해소를 돕는다. CBT(인지행동치료)의 인지 재구성 기법 및 감정 표현 글쓰기(expressive writing) 근거에 기반한다.

**입력 방식**
두 단계 입력:
1. 감정 태그 선택 (불안, 긴장, 화남, 슬픔, 외로움, 무기력, 감사, 안도 — 복수 선택 가능)
2. 자유 텍스트 메모장 입력 (글자 수 제한 없음)

'버리기' 버튼을 누르면 종이가 구겨지거나 불꽃이 튀는 애니메이션 후 텍스트가 화면에서 사라진다. 내용은 저장되지 않는 것이 원칙이며, 저장 여부는 사용자 선택으로 제공할 수 있다.

**표시 정보**
- 오늘 기록 여부 (완료 뱃지)
- 최근 7일 감정 태그 빈도 (어떤 감정이 자주 등장했는지 시각화)

**저장 정책**
버린 텍스트는 기본적으로 저장하지 않는다. 감정 태그만 날짜별로 저장하여 통계에 활용할 수 있다. 사용자가 원하면 일기 형식으로 보관 옵션 제공.

---

### 2-5. 운동 피로도 기록

**목적**
운동 후 주관적 피로도를 측정해 운동 강도가 적절한지, 회복이 충분한지 확인한다. 투석 트랙에서는 운동 금지 신호를 선제적으로 파악하는 용도로도 사용된다.

**입력 방식**
운동 완료 후 입력하도록 챌린지 완료 체크 후 팝업 또는 버튼 연동.
- 운동 종류 (걷기 / 자전거 / 근력 / 스트레칭 / 기타)
- 운동 시간 (분)
- 주관적 피로도: 1~5단계 이모지 스케일
  - 1 = 전혀 힘들지 않음
  - 2 = 약간 힘들었지만 문제없음
  - 3 = 적당히 힘들었음
  - 4 = 많이 힘들었음
  - 5 = 너무 힘들어서 중단할 뻔함
- 메모 선택 입력

**표시 정보**
- 오늘 운동 기록 요약
- 최근 7일 피로도 추이
- 피로도 4~5 연속 이틀 이상 시 "오늘은 가볍게 쉬어가는 것도 좋습니다" 문구 표시

**데이터 보존**
날짜별 저장. 하루 복수 운동 기록 가능. 최근 30일.

---

### 2-6. 병원 진료일 캘린더

**목적**
정기 진료, 투석 일정, 검사 예약을 한곳에서 관리한다. 알림 기능과 연동하여 진료 전날 리마인더를 보낸다.

**입력 방식**
날짜 선택 + 일정 종류 (정기 진료 / 투석 / 혈액검사 / 기타) + 병원명 / 메모 입력.

**표시 정보**
- 월별 캘린더 뷰 (일정 있는 날 도트 표시)
- 다음 진료일 카운트다운 ("다음 진료까지 D-3")
- 오늘 이후 일정 리스트 (최대 5건)
- 지난 일정 아카이브

**알림**
전날 오후 6시, 당일 오전 8시 알림 (설정에서 변경 가능).

---

### 2-7. 검사 수치 기록장

**목적**
병원 검사 결과를 날짜별로 입력하고 추세를 확인한다. 트랙에 따라 관리 항목이 다르다.

**트랙별 주요 수치**

| 트랙 | 주요 수치 |
|------|-----------|
| 투석 트랙 | K (칼륨), P (인), Hb (헤모글로빈), 투석 전후 체중 |
| 비투석 CKD | eGFR, 크레아티닌, 혈압 (수축기/이완기), 단백뇨 |
| 집중케어·일상케어 | 혈압, 혈당 (공복/식후), HbA1c, 콜레스테롤 (LDL/HDL) |
| 웰니스 | 혈압, 체중 (BMI 자동 계산 옵션), 콜레스테롤 |

**입력 방식**
검사 날짜 선택 + 항목별 수치 입력 (숫자 키패드). 항목은 트랙에 따라 사전 구성되어 있으나 사용자가 항목 추가/제거 가능.

**표시 정보**
- 항목별 최신값 + 이전 대비 증감
- 항목별 최근 5회 추이 꺾은선 차트
- 정상 범위 표시 선 (참고용, 의료적 판단 아님 명시)

**정상 범위 참고값 (표시 전용, 진단 목적 아님)**

| 수치 | 참고 범위 | 출처 근거 |
|------|-----------|-----------|
| K (칼륨) | 3.5~5.0 mEq/L | KDIGO 2024 |
| P (인) | 2.5~4.5 mg/dL | KDIGO 2024 |
| Hb | 남 13.5~17.5 / 여 12.0~16.0 g/dL | WHO |
| eGFR | ≥60 정상 | KDIGO CKD 분류 |
| 크레아티닌 | 남 0.7~1.2 / 여 0.5~1.0 mg/dL | 일반 참고치 |
| 혈압 | 수축기 <130 / 이완기 <80 mmHg | AHA 2023 |
| 공복 혈당 | 70~100 mg/dL | ADA 2024 |
| LDL 콜레스테롤 | <100 mg/dL (고위험군) | ACC/AHA |

**데이터 내보내기**
CSV 또는 PDF로 내보내기 기능 제공. 진료 시 의료진에게 제출 가능한 형식.

## 3. 정보 구조 (IA) — 탭 구성

현재 앱의 5탭(오늘, 기록, 통계, 보상, 설정)에 기록 기능을 통합하는 방향으로 설계한다.

```
앱 하단 탭
├── 오늘       : 챌린지 체크리스트 + 일일 트래킹 빠른 입력
├── 기록       : 수분 / 체중 / 수면 / 감정 / 피로도 상세 입력 및 내역
├── 통계       : 챌린지 완료율 차트 + 수치 추세
├── 케어       : 검사 수치 기록장 + 병원 캘린더  ← 신규
└── 설정       : 알림, 트랙, 목표값, 데이터 관리
```

보상 탭은 통계 탭 하위로 흡수하거나 별도 유지한다. 탭이 6개 이상이 되면 하단 탭바가 혼잡해지므로, 케어 탭 신설과 동시에 보상을 설정 또는 통계 탭 내 서브 섹션으로 이동하는 것을 권장한다.

## 4. 오늘 탭과 기록 기능의 연계 방식

일일 트래킹 5개 항목은 오늘 탭 하단에 빠른 입력 카드로 노출한다. 각 카드는 오늘 입력 완료 여부를 즉시 보여주고, 탭하면 기록 탭의 해당 섹션으로 이동하거나 인라인 입력 시트가 열린다.

```
오늘 탭 하단 — 오늘의 기록
┌──────────────┬──────────────┐
│ 💧 수분       │ ⚖️ 체중       │
│ 0 / 2,000mL  │ 미입력       │
├──────────────┼──────────────┤
│ 🌙 수면       │ 😮‍💨 스트레스  │
│ 7시간 30분   │ 미입력       │
├──────────────┴──────────────┤
│ 🏃 운동 피로도                │
│ 오늘 운동 기록 없음           │
└──────────────────────────────┘
```

## 5. 데이터 모델 요약

```
daily_log {
  date: string (YYYY-MM-DD)
  water: [ { time, ml, type } ]
  weight: { kg, time, note }
  sleep: { bed_time, wake_time, duration_min, wake_count }
  stress: { tags: string[], saved_note?: string }
  exercise: [ { type, duration_min, fatigue_level, note } ]
}

archive {
  lab_records: [ { date, track, values: { [key]: number } } ]
  appointments: [ { date, type, hospital, note, reminded: boolean } ]
}

user_settings {
  track, stage
  water_goal_ml
  weight_alert_kg
  notif_on, notif_time
  evening_on, evening_time
}
```

## 6. 구현 우선순위

트랙에 관계없이 모든 사용자에게 해당하고 매일 사용 빈도가 높은 기능을 먼저 개발한다.

**1순위 (필수, 매일 사용)**
- 수분 섭취 기록
- 체중 기록
- 수면 기록

**2순위 (차별화, 정서적 가치)**
- 스트레스 감정 쓰레기통
- 운동 피로도 기록

**3순위 (아카이브, 의료 연계)**
- 검사 수치 기록장
- 병원 진료일 캘린더

## 7. 주의 사항 및 면책 기준

정상 범위 표시에는 반드시 "참고용 수치이며 의료적 진단을 대체하지 않습니다. 수치 해석은 담당 의료진에게 문의하세요." 문구를 명시한다. 경고 표시(체중 급증, 수분 초과 등)도 동일하게 "이상 시 의료진에게 연락하세요" 안내와 함께 표시한다.
